Overview of Git and GitHub

Git is usually used for sharing, collaboration and version control. GitHub is a platform for hosting code. A software that runs locally is Git. All our folders are in the computer's storage. To share, collaborate and save various versions we can use Github as an online platform. Git is useful if two people work on the same file but different sections they can upload it on git and it will merge automatically. Having a platform to exchange the updates and receive the changes from others helps us to easily and effectively collaborate. Git can be accessed in two ways. One is through the command line (Windows) or terminal (Mac) where commands must be manually typed in. The other way is through a desktop app with a GUI.

How to install Git and GitHub

Git can be easily installed on any computer but most of the macs and linux come with default installation.

Checking for Git

Open your terminal to check if you already have Git installed on your PC.

If you are on a Mac, the command prompt application goes by the name of ‘Terminal’.

If you are on Windows, you can either open ‘Git Bash’ or Windows Command Prompt.

Once the terminal application has been opened, just type in ‘git version’. If the version of Git appears on screen, it means that Git is already installed on your PC. Alternatively, if an alert saying git is an unknown command appears on the terminal, then Git has not yet been installed on your PC.

Git config

Git config is a command that is used to set the username and email while making commits. Git config can be used only once. (Replace and with any name that you want.)

Usage: git config –global user.name “{name/}”

Usage: git config –global user.email “”

Git clone

This command is used to get a copy of your repository on your PC.

Git add

Git add is a command used to add a particular file to the staging area before it is committed. For instance, if your workspace has 20 files, and you want to only update 3 of those files, you can use ‘git add ’ to add that particular file to the staging area.

If you want to add all the files to the staging area, then you can use the command ‘git add .’

Git commit

Git commit is the command used to store a particular version of your file in the history. You can think of a git commit as a snapshot that you can always roll back or refer to later, even after making further modifications.

Use the command ‘git commit -m “”’ to store a snapshot of the file with the specific ‘message’

If you want to commit the added files only (the ones you added using git add), use the command ‘git commit -a’

Git push

Git push is a command that formally sends all your commits to the remotely connected repository. For example, if you have a repository on Github that is connected via git, git push will update all the corresponding files there.

Git status

Show important information, on what branch you are on, files that are on the working conditions.

Git pull

This command is used to get the changes made on the repository on the local machine or server.

How to resolve conflicts

Creating pull requests typically compared the source code with the update, and also the destination with that of the original code present. Typically, when working with multiple members in a team, chances are that other team members inadvertently made changes to the code. A conflict is just a notification of this mismatch in the code when you are pulling the updated files.

We have a conflict now, if the changes are correct the click on accept all incoming changes.

Branching

Git allows working on different versions of a repository with the help of branches. It helps you diverge from the main development line, and make appropriate changes without affecting the main line. Once you are happy with the changes made, the secondary ‘branch’ can be merged with the main ‘branch’.

Git log

Shows the log of all commits. The log output is present in a reverse chronological order.

Repository

To organize a single project we usually use a repository. It usually has project information or a README file. It contains everything necessary for the project like data sets, files, spreadsheets, Images and videos.
